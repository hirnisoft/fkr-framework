# FKR – Functional Complexity Reduction

> Reduce complexity. Not manage it.

Most systems don't fail because they are too simple.  
They fail because they accumulate unnecessary complexity.

FKR is a system design approach that eliminates that complexity at its source.

by Roger Sandmeier

---

## Definition

> Functional Complexity Reduction (FKR) is the practice of eliminating unnecessary complexity to reduce decisions, mental load, and system friction.

---

## What is FKR?

Functional Complexity Reduction (FKR) removes unnecessary complexity at its source.

It does not optimize complexity.  
It eliminates it.

> Not: “How can this be managed better?”  
> But: “Why does this exist at all?”

---

## Core Idea

FKR shifts the focus from optimization to elimination.

- reduce decisions  
- reduce redundancy  
- reduce mental load  
- build systems that are clear and stable  

> Clarity is not achieved by organizing complexity.  
> Clarity is achieved by removing it.

---

## Principles

- **Function over features**  
  Every element has a clear primary purpose.

- **Fewer options**  
  More choice creates more friction.

- **External systems**  
  The system holds information – not your head.

- **Structured behavior**  
  Systems replace ad-hoc decisions.

---

## What FKR is NOT

- not minimalism  
- not productivity hacking  
- not self-optimization  

FKR is:

> System design for everyday life, decisions, and structures.

---

## Scope

This repository defines:

- core principles and laws  
- system design patterns  
- anti-patterns and failure modes  
- real-world applications  

FKR is not a tool.  
It is a way to design systems.

---

## Guiding Rule

> Eliminate before you optimize.
