# Functional Complexity Reduction (FKR)

## Classification

FKR does not only apply to technical systems.

In the context of FKR, everything is a system:

- things you own  
- spaces you use  
- processes you repeat  
- decisions you make  

All of these can be either clear – or complex.

---

## Definition

Functional Complexity Reduction is an approach to designing everyday life, systems, and decisions in which unnecessary complexity is removed and necessary complexity is clearly structured.

The goal is maximum clarity with minimal mental load.

---

## Core Principle

> Complexity is not avoided, but consciously designed.

---

## The 4 Pillars

### 1. Functional Clarity

Each element has a primary, clearly recognizable function.

No unnecessary overlaps.  
No unclear responsibilities.

---

### 2. Reduced Options

The number of possible choices is actively limited.

Goal:

- faster decisions  
- less mental load  
- consistent outcomes  

---

### 3. External Systems

Information and tasks are consistently externalized.

Principle:

- mind = thinking  
- system = storage  

---

### 4. Structured Systems

Recurring processes are standardized.

Systems replace:

- improvisation  
- randomness  
- ad-hoc decisions  

---

## Decision Filter (FKR Check)

For each new element:

1. Does it have a clear function?  
2. Does it replace something existing or duplicate complexity?  
3. Does it reduce decisions or increase them?  
4. Does it fit into an existing system?  

If ≥1 answer is critical → do not integrate.

---

## Anti-Patterns

Typical mistakes:

- “can also…” objects  
- multiple similar solutions for the same problem  
- things without a clear place or state  
- knowledge only in the head  
- systems that are not used  

---

## Target State

- clear responsibilities per element  
- minimal decision load  
- understandable systems  
- controlled instead of random complexity  

---

## Summary

> Clear function. Fewer options. External systems.  
> Complexity is designed, not accumulated.
